# Existing product extraction

## Read

- README and product brief
- route definitions
- relevant page/component files
- global CSS and theme tokens
- logo, icon, screenshot, and public asset folders
- realistic product copy and data examples

## Run when possible

Use a browser to inspect the actual product. Capture the strongest product states at useful dimensions. Record whether each screenshot is real, mocked from real UI, or generated.

## Evidence table

| Evidence | Source | Marketing use | Confidence |
|---|---|---|---|
| Product category | README + routes | Hero positioning | High |
| Main workflow | Live app | Hero/demo | High |
| User role | Auth/navigation | Audience copy | High |
| Metric | Unknown | Forbidden | None |

## Landing-page translation

Do not copy operational density directly. Convert workflows into a narrative:

1. Problem or outcome
2. Product proof
3. How it works
4. Differentiated capability
5. Trust or process evidence
6. CTA
